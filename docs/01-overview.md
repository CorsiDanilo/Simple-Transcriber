# 01 - Project Overview 🌍

## Introduction

**Transcriber** is a modern Android application designed to bridge the gap between heavy cloud-based AI and privacy-focused local processing. Its primary goal is to provide users with a reliable way to convert audio to text, refine that text for better readability, and manage a history of transcriptions.

## Core Philosophy

1. **User Choice**: Users should decide where their data is processed.
2. **Seamless Experience**: Transcription should be fast and reliable, whether online or offline.
3. **Accessibility**: A simple, clean interface that follows modern Android design guidelines (Material 3).
4. **Maintenance**: Built with a clean architecture to allow easy integration of new AI models or engines.

## Key Use Cases

- **Meeting Notes**: Transcribing recorded meetings and using "Refinement" to clean up the transcript.
- **Privacy-Sensitive Content**: Using LiteRT models to ensure audio is never uploaded to the cloud.
- **Fast Dictation**: Sharing audio messages from other apps (like WhatsApp or Telegram) to Transcriber for quick reading.

## Future Roadmap

- [ ] **Real-time streaming**: Transcribing audio as it is being recorded.
- [ ] **Speaker Diarization**: Identifying different speakers in the audio.
- [ ] **Expanded Model Support**: Integration with more on-device AI frameworks (like AICore).
- [ ] **Export Options**: Exporting transcripts to PDF, TXT, or Markdown.
